Astropay Card - Cashouts

Introduction

Merchant will be able to create an AstroPay Card with the choosen value. The cost of the card plus a fee will be deducted from merchans account.
Once a new Cashout Card is requested, the new card will be issued and sent to customer's email.

Considerations

Technical details

  • Communication between Merchant and AstroPay should be made by POST.
  • The protocol used is purely HTTPS.
  • Sensible information is sent server to server.
  • The return format of the functions is json.

Security details

  • Credentials must be used in all communications.
  • Both parties must use an HMAC-SHA-256 (RFC 2104) code to verify the integrity of the information received from each side.
  • HMAC‐SHA‐256 code (32 bytes) is generated using a key provided by AstroPay. This code is converted to string as a hexadecimal representation of the code (64 upper chars).
  • All communications between Merchant and AstroPay should be made exclusively through registered IPs on both sides.

How to start?

Create your account in our site https://merchant.astropaycard.com/sign_up. After the registration is finished you will have access to your credentials, your account configuration and your backoffice at https://www.merchant.astropaycard.com.

Remember: register your server IP in the panel https://www.merchant.astropaycard.com in section Integration->Credentials & Settings

Environments

Your account is associated with two different environments: Sandbox and Live.
Sandbox: Test environment available for integration development and testing which simulates the requests and transactions available in the platform. On test environment no transactions will actually be processed.
Live: Production environment.

URLs, credentials & account configuration

Base URL for communication:

Sandbox

https://sandbox-api.astropaycard.com/cashOut/

Live

https://api.astropaycard.com/cashOut/

Note: You will find your credentials in the panel: "Integration -> Credentials & Settings". Notice that credentials are different between both environments.
Note: Once the integration is finished and tested, fill your account details in the panel: "Settings -> Merchant Account" and "Request to go live". Your live credentials will be available when Merchant's account is in Live mode.

Function: sendCard

This function is used to generate a Cashout Card.

POST:
https://api.astropaycard.com/cashOut/sendCard

Request

Mandatory parameters

Field Format Description Example
x_login String (length: 10 chars) Your merchant ID in Astropay platform AsGsd35Grf 1
x_trans_key String (length: 10 chars) Your merchant password in Astropay platform D23weF2f4g 1
x_amount Decimal (max. 2 decimal numbers) Cashout amount 100.95
x_currency String (length: 3 chars) Cashout currency code EUR 2
x_email String (max. length: 100 chars) User email address astropay@astropaycard.com
x_control String (length: 3 chars) Control string calculated by you EUR 3

1 x_login and x_trans_key are your credentials. Remember to find them in the panel, section Integration -> Credentials & Settings.

2 See available currencies

3 Control string

  • $secret - Merchant's account secret key
  • $x_amount - Cashout amount
  • $x_currency - Cashout currency code
  • $x_email - User email address

The control string is calculated, as follows:

$control_string = sha1 ( $this->merchant->secret . number_format ( $this->params->x_amount, 2 ) . $this->params->x_currency . $this->params->x_email );

Optional parameters

Field Format Description Example Default
x_window Integer The time window after a transaction is taken as duplicated 60 10 1
x_reference String (length: 10 chars) Your internal cashout reference cashout_123 -

1 Transactions sent with same amount and same card information, within a time window of 10 seconds will be taken as duplicated. Batch transactions not recommended.

Response

Field Description
code Response code 200 if success. If not see errors
message Response message "success" or error message
response Response of request enum('SUCCESS','ERROR')
id_cashout Astropay internal ID "success" or error message
auth_code Astropay authentication code Use for VOID a cashout (not available)
email User email Value sent in request
amount Cashout amount Value sent in request
currency Cashout currency Value sent in request
control Control signature 1
1 Control signature
  • $secret - secret key of the account
  • $id_cashout - identificator of cashout
  • $x_email - user email
  • $x_amount - cashout amount
  • $x_currency - cashout currency

The control signature is calculated, as follows:

$control_string = sha1($secret.$id_cashout.$x_email.number_format($x_amount,2).$x_currency)
Notes:
  • Once this method is invoked, the cashout amount is automatically discounted from Merchant's balance.
  • It is Merchant's responsibility to send the correct destination email address. We suggest asking for a confirmation after the user enters the email address.
  • All cards are generated in USD (dollar). If a different currency is sent amount will be converted to USD.
